Accounts & API Keys

// the fuel that powers your agents

API keys are your agents' identity cards. They allow code to call Claude, GPT, and other services. Treat them like passwords ‚Äî a leaked key can rack up thousands of dollars in charges within hours.

‚ö† Security Warning
Never commit API keys to GitHub. Never paste them in chat messages. Store them in environment variables or a secrets manager. A single leaked key can result in thousands of dollars in unauthorized charges overnight.

Virtual Environments

// isolated python worlds ‚Äî free tools only

A virtual environment is an isolated Python installation for your project. It keeps packages separate between projects so nothing conflicts ‚Äî crucial when you have a CrewAI crew, a LangGraph workflow, and a personal tool all needing different library versions.

‚ú¶ Golden Rule
Always activate a virtual environment before installing Python packages. Installing globally with pip pollutes your system Python and causes version conflicts that are painful to debug. Every project gets its own env.

Four Free Tools ‚Äî Pick Your Style

venv
Built-in
python3 -m venv .venv
Comes with Python. Zero install. One env per folder. Simple and reliable ‚Äî great for beginners.
uv
10-100√ó Faster
curl -LsSf https://astral.sh/uv/install.sh | sh
Blazing-fast Rust-based pip replacement. Drop-in venv + package manager. Highly recommended for agent work where you install packages often.
pyenv + venv
Recommended
brew install pyenv
Manage multiple Python versions (3.10, 3.11, 3.12) and create envs per project. Best for juggling different projects with different Python needs.
conda / miniconda
Popular
brew install --cask miniconda
Full environment manager with non-Python dependencies. Great if you mix Python with data science or C libraries. Heavier than venv but powerful.

Recommended Setup for This Guide

We recommend uv for speed + pyenv if you need multiple Python versions. For beginners, built-in venv is perfectly fine.

BASH ‚Äî Quick Start with venv (simplest)
# Create your agent company project folder mkdir -p ~/Projects/AgentCompany && cd ~/Projects/AgentCompany # Create environment python3 -m venv .venv # Activate (run this every new Terminal session) source .venv/bin/activate # You'll see (.venv) prefix in your prompt ‚Äî you're inside! # Now install packages safely pip install crewai crewai-tools # Deactivate when done deactivate
BASH ‚Äî Quick Start with uv (fastest)
# Install uv curl -LsSf https://astral.sh/uv/install.sh | sh source ~/.zshrc # Create project with venv in one command uv init ~/Projects/AgentCompany cd ~/Projects/AgentCompany # Add packages (10-100x faster than pip) uv add crewai crewai-tools langchain-anthropic # Run scripts inside env automatically uv run python main.py
BASH ‚Äî Auto-activate env on folder entry (optional quality of life)
# Install direnv for automatic activation brew install direnv # Add to ~/.zshrc: eval "$(direnv hook zsh)" # In your project folder, create .envrc: echo "source .venv/bin/activate" > .envrc direnv allow # Now the env activates automatically when you cd into the folder!

Build Your Agent Team

// hire your first AI employees

Now we build the actual agents. We'll create a CrewAI crew with three agents ‚Äî PM, Developer, and Tester. This is your Minimum Viable Company.

BASH ‚Äî Project structure
mkdir -p ~/Projects/AgentCompany/{agents,tasks,tools,config,outputs} cd ~/Projects/AgentCompany touch main.py
PYTHON ‚Äî main.py (3-agent crew)
from crewai import Agent, Task, Crew, Process from langchain_anthropic import ChatAnthropic from langchain_openai import ChatOpenAI claude_opus = ChatAnthropic(model="claude-opus-4-6") claude_sonnet = ChatAnthropic(model="claude-sonnet-4-6") gpt4o = ChatOpenAI(model="gpt-4o") pm = Agent( role="Project Manager", goal="Break down requirements into clear tasks and coordinate the team", backstory="10-year veteran PM who ships on time, every time.", llm=claude_opus, verbose=True ) developer = Agent( role="iOS Developer", goal="Write clean, production-ready Swift code that passes all tests", backstory="Expert Swift developer who loves elegant, testable code.", llm=claude_sonnet, verbose=True ) tester = Agent( role="QA Engineer", goal="Ensure all code is tested and working before deployment", backstory="Meticulous QA engineer who finds bugs before users do.", llm=claude_sonnet, verbose=True ) plan_task = Task(description="Break down the iOS feature into developer tasks", expected_output="Specific tasks with acceptance criteria", agent=pm) dev_task = Task(description="Implement the feature based on PM's breakdown", expected_output="Working Swift code with documentation", agent=developer, context=[plan_task]) test_task = Task(description="Write and run XCTest tests for the feature", expected_output="Test results with pass/fail and coverage", agent=tester, context=[dev_task]) crew = Crew(agents=[pm, developer, tester], tasks=[plan_task, dev_task, test_task], process=Process.sequential, memory=True, verbose=True) result = crew.kickoff(inputs={"feature": "Add a favorites list with persistence"}) print(result)
BASH ‚Äî Run your crew
source ~/Projects/AgentCompany/.venv/bin/activate cd ~/Projects/AgentCompany python main.py

CI/CD Automation

// ship code without touching a button

CI/CD is the pipeline that takes your code from a commit all the way to TestFlight automatically. Your CI/CD agent watches for triggers ‚Äî passing tests, a PR merge ‚Äî and fires the next step.

YAML ‚Äî .github/workflows/ios-deploy.yml
name: iOS Agent Pipeline on: push: branches: [main] jobs: test-and-deploy: runs-on: macos-latest steps: - uses: actions/checkout@v4 - name: Run Tests run: | xcodebuild test \ -scheme MyApp \ -destination 'platform=iOS Simulator,name=iPhone 15' - name: Notify Agent via Webhook if: success() run: | curl -X POST ${{ secrets.N8N_WEBHOOK_URL }} \ -H "Content-Type: application/json" \ -d '{"event": "tests_passed", "branch": "${{ github.ref }}"}' - name: Deploy to TestFlight if: github.ref == 'refs/heads/main' && success() run: | gem install fastlane fastlane beta
BASH ‚Äî Set up Fastlane
cd ~/Projects/MyiOSApp gem install fastlane fastlane init # Choose option 2: Automate beta distribution to TestFlight
